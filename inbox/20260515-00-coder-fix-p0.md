## 任务二：P0-2 — pytest.ini 补充本地覆盖率配置

### 修改文件

```
backend/pytest.ini
```

### 任务说明

**背景澄清**：CI（`.github/workflows/ci.yml` line 202）已通过命令行参数强制了 `--cov-fail-under=50`，门控本身未失效。本次修复目标是让**本地 `pytest tests/`** 也自动显示覆盖率摘要，对齐 CLAUDE.md 中的验证命令规范。

**不要**在 `addopts` 加 `--cov-fail-under=50`（会导致每次本地开发都因覆盖率不足报失败，干扰日常调试）。

**修改方法**：

将：
```ini
addopts = -v --tb=short --import-mode=importlib
```

改为：
```ini
addopts = -v --tb=short --import-mode=importlib --cov=services --cov=app --cov-report=term-missing
```

### 验证

修改后本地执行：
```bash
cd backend && pytest tests/ -q --no-header 2>&1 | tail -5
```

预期输出末尾包含 `TOTAL ... %` 的覆盖率行，即为成功。  
（`pytest-cov` 已在 `requirements.txt` 中，无需额外安装）
