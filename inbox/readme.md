# 📥 Agent Inbox (工作缓存与草稿区)

## 📌 定位与用途
本目录 `mulan-bi-platform/inbox` 是专为 AI Agents 设立的“临时工作台”。
仅用于存放工作时的**缓存文件、阶段性交接文件、思维链临时产出、以及任务跟踪清单**。
> **注意：** 本目录下的所有文件均被视为临时草稿，未来会定期统一清理删除。

## 🚫 严格禁止（核心红线）
为了维护项目的“单一事实来源 (Single Source of Truth)”，**严禁将任何正式的项目资产写入本目录**。

绝对禁止放入本目录的内容包括但不限于：
- ❌ **正式规格文档**：`PRD.md`、`SPEC.md` 等（必须存入 `docs/prd/` 或 `docs/specs/`）
- ❌ **项目源代码**：任何前后端代码如 `.ts`, `.tsx`, `.py`, 等（必须存入对应工程目录）
- ❌ **正式测试报告**：如 `TESTER_PASS.md`（必须存入 `docs/tests/`）

## 📝 格式与命名规范
为了方便知识管理工具（如 Obsidian）进行索引，请遵循以下规范：
1. **文件格式**：默认使用 Markdown (`.md`) 格式存储。
2. **命名规则**：所有文件必须以 `yyyymmdd-hh` 的前缀开头，以记录生成时间。
   - *示例：`20231025-14-coder-thinking-process.md`*
