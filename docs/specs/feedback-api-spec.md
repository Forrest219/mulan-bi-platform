## 1. 背景

用户在查看 AI 回答后，可通过点赞（👍）或踩（👎）提供反馈。
反馈数据写入 `message_feedback` 表，供运营和模型改进使用。

---

## 2. 数据库表

### 2.1 表定义

```sql
CREATE TABLE message_feedback (
    id SERIAL PRIMARY KEY,
    user_id INTEGER NOT NULL REFERENCES users(id),
    username VARCHAR NOT NULL,
    conversation_id VARCHAR,
    message_index INTEGER,
    question TEXT,
    answer_summary TEXT,
    rating VARCHAR(4) NOT NULL CHECK (rating IN ('up', 'down')),
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 2.2 字段说明

| 字段 | 类型 | 说明 |
|-----|------|------|
| `id` | SERIAL PK | 自增主键 |
| `user_id` | INTEGER FK | 来自 JWT 解析，不由前端传入 |
| `username` | VARCHAR | 来自 JWT 解析，不由前端传入 |
| `conversation_id` | VARCHAR NULL | 对话 ID，可为 null（无多轮上下文时） |
| `message_index` | INTEGER NULL | 消息在对话中的位置索引（从 0 开始） |
| `question` | TEXT NULL | 该 AI 消息对应的用户问题原文 |
| `answer_summary` | TEXT NULL | AI 回答前 100 字（由前端截取） |
| `rating` | VARCHAR(4) | `'up'` 或 `'down'`，有 CHECK 约束 |
| `created_at` | TIMESTAMP | 写入时间，默认 NOW() |

## 3. 接口定义

### 3.1 基本信息

| 项 | 值 |
|----|---|
| Method | POST |
| Path | `/api/feedback` |
| Auth | 需要登录（JWT Cookie 或 Authorization Header） |
| Content-Type | `application/json` |

### 3.2 Request Body（前端传）

```json
{
  "conversation_id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "message_index": 2,
  "question": "各区域销售额是多少？",
  "answer_summary": "华东区域销售额最高，为 1,234 万元；华北次之...",
  "rating": "up"
}
```

| 字段 | 类型 | 必填 | 说明 |
|-----|------|------|------|
| `conversation_id` | string \| null | 否 | 当前对话 ID；无多轮上下文时传 null |
| `message_index` | integer | 否 | AI 消息在 `streamingMessages` 数组中的索引 |
| `question` | string | 否 | 对应用户问题（取本条 AI 消息前一条 user 消息） |
| `answer_summary` | string | 否 | `msg.content.slice(0, 100)`，由前端截取 |
| `rating` | `"up"` \| `"down"` | 是 | 点赞或踩 |

**注意**：`user_id` 和 `username` 不由前端传入，后端从 JWT 中解析。

### 3.3 Response

成功（HTTP 200）：
```json
{"ok": true}
```

失败：
- `401 Unauthorized`：未登录
- `422 Unprocessable Entity`：`rating` 不合法（非 `"up"` / `"down"`）
- `500 Internal Server Error`：数据库写入失败

## 4. 后端实现

### 4.1 新建文件：`backend/app/api/feedback.py`

```python
"""消息反馈 API — POST /api/feedback"""
import logging
from datetime import datetime, timezone

from fastapi import APIRouter, Request
from pydantic import BaseModel, field_validator
from sqlalchemy import text
from typing import Optional

from app.core.database import get_db
from app.core.dependencies import get_current_user

logger = logging.getLogger(__name__)
router = APIRouter()


class FeedbackRequest(BaseModel):
    conversation_id: Optional[str] = None
    message_index: Optional[int] = None
    question: Optional[str] = None
    answer_summary: Optional[str] = None
    rating: str  # 'up' | 'down'

    @field_validator('rating')
    @classmethod
    def validate_rating(cls, v: str) -> str:
        if v not in ('up', 'down'):
            raise ValueError("rating 必须为 'up' 或 'down'")
        return v


@router.post("/feedback")
async def submit_feedback(body: FeedbackRequest, request: Request):
    """
    POST /api/feedback — 提交消息反馈

    user_id 和 username 从 JWT 解析，不接受前端传入。
    """
    db = next(get_db())
    user = get_current_user(request=request, db=db)
    user_id = user.get("id")
    username = user.get("username") or user.get("email") or ""

    try:
        db.execute(
            text(
                "INSERT INTO message_feedback "
                "(user_id, username, conversation_id, message_index, question, answer_summary, rating, created_at) "
                "VALUES (:user_id, :username, :conversation_id, :message_index, :question, :answer_summary, :rating, :created_at)"
            ),
            {
                "user_id": user_id,
                "username": username,
                "conversation_id": body.conversation_id,
                "message_index": body.message_index,
                "question": body.question,
                "answer_summary": body.answer_summary[:100] if body.answer_summary else None,
                "rating": body.rating,
                "created_at": datetime.now(timezone.utc),
            },
        )
        db.commit()
        logger.info(
            "反馈已记录 user_id=%s rating=%s conversation_id=%s",
            user_id, body.rating, body.conversation_id,
        )
        return {"ok": True}
    except Exception as exc:
        db.rollback()
        logger.error("写入反馈失败: %s", exc, exc_info=True)
        raise
    finally:
        db.close()
```

### 4.2 在 main.py 注册路由

**文件**：`backend/app/main.py`

在现有 import 区块末尾（`from app.api import ...` 之后）追加：
```python
from app.api import feedback
```

在现有路由注册区块末尾追加（`app.include_router(chat.router)` 后面）：
```python
app.include_router(feedback.router, prefix="/api", tags=["反馈"])
```

**注意**：`prefix="/api"` 已在路由注册时指定，`feedback.py` 内部路由为 `/feedback`，最终合并为 `/api/feedback`。

## 6. 安全约束

| 约束 | 说明 |
|-----|------|
| user_id 不由前端传 | 后端从 JWT 解析，防止伪造 |
| answer_summary 强制截断 | 后端二次截断至 100 字，防止超长内容 |
| rating CHECK 约束 | 数据库层和 Pydantic 双重校验 |
| 登录态校验 | `get_current_user()` 若未登录抛 401 |

## 7. 验收口径

| 测试项 | 操作 | 预期结果 |
|-------|------|---------|
| 未登录拒绝 | 不携带 token 调用 `POST /api/feedback` | 返回 401 |
| 正常写入 | 登录后调用，body 含合法 rating | 返回 `{"ok": true}`，`message_feedback` 表新增记录 |
| user_id 注入 | 检查数据库记录 | `user_id` 与登录用户一致，非前端传入 |
| rating 校验 | 传 `rating: "maybe"` | 返回 422 |
| 无 conversation_id | `conversation_id: null` | 正常写入，该字段为 NULL |
