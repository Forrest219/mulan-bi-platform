## 1. 模块定位

**数据库监控模块**是 Mulan BI Platform 的核心扫描引擎，负责连接用户的数据源、执行 DDL 规范检查、生成治理报告。是平台"数据质量保障"能力的基础入口。

**核心价值**：帮助 BI 团队系统性发现存量数据库的表结构问题，形成治理闭环。

## 3. 功能需求

### 3.1 数据源管理

**3.1.1 添加数据源**
- 支持数据库类型：MySQL、SQL Server、PostgreSQL、Hive、StarRocks、Doris
- 配置字段：
  - 数据源名称（业务命名，如"ODS 层 MySQL"）
  - 数据库类型（dropdown）
  - Host / Port / Database
  - 用户名 / 密码（加密存储）
  - 连接超时时间
- 连接参数（SSL、编码等）作为可选高级选项

**3.1.2 数据源列表**
- 展示所有已配置数据源：名称、类型、连接状态、最近扫描时间、问题数
- 支持搜索、筛选（按类型、按状态）
- 支持编辑、删除（需二次确认）

**3.1.3 连接测试**
- 在保存前验证连接是否可用
- 显示具体错误信息（网络不通、密码错误、库不存在等）

### 3.2 规则扫描

**3.2.1 执行扫描**
- 支持按数据源批量扫描（所有表）
- 支持按单个或多个指定表名扫描
- 扫描时显示实时进度（已扫描 / 总表数）
- 支持中断扫描

**3.2.2 扫描历史**
- 每次扫描记录：执行时间、数据源、耗时、问题统计
- 支持按时间范围筛选
- 支持重新查看历史报告

### 3.3 扫描结果

**3.3.1 问题列表**
- 按表聚合显示问题，支持钻取到列级详情
- 问题分级：High / Medium / Low（对应 error / warning / info）
- 支持按级别、按规则类型、按表名筛选
- 问题字段：表名、列名（可选）、规则名称、级别、描述、修复建议

**3.3.2 问题趋势**
- 统计当前数据源所有问题数、高/中/低分布
- 提供问题随时间变化的趋势图（最近 5 次扫描对比）

**3.3.3 报告导出**
- 支持导出为 HTML 报告（可分享）
- 报告包含：执行摘要、问题分布、各表详情

### 3.4 规则配置（关联）

- 各数据源可绑定不同的 `rules.yaml` 配置文件
- 支持从预置规则模板中选择（MySQL 规范 / Hive 规范 / StarRocks 规范）
- 管理员可在规则配置页面维护规则内容

## 4. 非功能需求

### 4.1 性能
- 单次扫描 MySQL 1000 张表，耗时不超过 60 秒
- 扫描过程中数据库连接保持活跃，不因超时断开
- 大结果集分页展示，不一次性加载

### 4.2 安全
- 数据库密码加密存储（不可逆加密）
- 不在日志中打印密码
- 数据源连接凭证仅管理员可见

### 4.3 可用性
- 连接失败时给出明确错误信息
- 扫描中断不污染数据库状态（只读操作）
- 页面加载时显示骨架屏，不出现白屏

## 8. 技术实现要点

### 8.1 数据库连接
- 各数据库类型使用对应驱动（PyMySQL / pymssql / pyhive / pymysql）
- 连接池管理，避免频繁建立/断开
- 密码使用 `Fernet` 对称加密存储

### 8.2 扫描执行
- 扫描任务为同步 API（限时 5 分钟），大规模扫描走后台任务队列（Celery / RQ）
- 扫描过程只读，不修改目标数据库任何内容
- 断点续扫：记录已扫描表，失败后可跳过已成功部分

### 8.3 前端状态管理
- 使用 React Context 或 Zustand 管理数据源列表和扫描状态
- 扫描进度使用轮询（每 2 秒拉取一次状态）

## 11. 用户管理模块改造计划

### 11.1 背景

现有用户体系为 admin/user 二级，无法支撑上述四角色分工。需要扩展为完整的角色体系。

### 11.2 改造范围

**数据库变更：**
- `users` 表新增 `role` 字段（ENUM: admin / data_admin / analyst / user）
- `data_sources` 表新增 `owner_id` 字段（关联用户，标记创建者）

**后端改动：**
- `AuthService` 增加角色定义常量
- `authenticate()` 登录返回用户角色
- `hasPermission()` 改为基于角色的权限判断
- 数据源 API 按 `owner_id` 过滤非管理员的查询结果

**前端改动：**
- 用户管理列表页：增加「角色」列，支持筛选
- 用户编辑/创建弹窗：增加「角色」下拉选择
- `AuthContext` 增加 `role` 字段
- `ProtectedRoute` 增加基于角色的访问控制

**API 端点变更：**
- `PUT /api/users/{id}/role` — 更新用户角色（管理员专属）
- `GET /api/datasources` — 按 `owner_id` 过滤（业务分析师只看自己创建的）

### 11.3 实施顺序

| 阶段 | 内容 | 优先级 |
|------|------|--------|
| 阶段一 | 数据库加字段、后端角色判断、前端角色下拉 | P0 |
| 阶段二 | 数据源 owner 隔离（普通用户看不到别人的数据源） | P1 |
| 阶段三 | 业务分析师只读视图（只能看报告） | P1 |
