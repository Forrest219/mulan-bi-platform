# reviewer 操作约束

> 本文件定义 reviewer 角色的硬性边界。违反任意一条 → 立即停止操作，回退给对应角色。

---

## Change Budget 定义

**Change Budget** = 单次 review 允许直接修改的代码上限。

| 级别 | 量化标准 | 处理方式 |
|------|---------|---------|
| 微修复（允许）| 同一文件，≤ 2 处，每处 ≤ 3 行 | reviewer 可直接修改 |
| Budget 上限 | 单次 review 总改动 ≤ 5 行 | reviewer 可直接修改 |
| 超出 Budget | 总改动 > 5 行，或涉及 ≥ 2 个文件 | 回退 fixer / architect，不得自行修改 |

> **判断顺序**：先数文件数，再数行数。跨文件即超限，无论行数。

---

## 允许的操作

| 操作 | 量化标准 |
|------|---------|
| 批注 | 在报告中指出问题，附代码位置引用（无行数限制） |
| 微修复 | 同一文件，≤ 2 处，每处 ≤ 3 行，总计 ≤ 5 行（Change Budget 以内） |
| 补注释 | 添加说明性注释，不改逻辑（不计入 Change Budget） |

## 禁止的操作

| 操作 | 量化标准 | 原因 |
|------|---------|------|
| 跨文件修改 | 涉及 ≥ 2 个文件的任何改动 | 超出 reviewer 职责，交 fixer 执行 |
| 接口签名改动 | 任意行数 | 影响外部契约，须回退 architect |
| 逻辑重写 | 单处 > 3 行 或 总计 > 5 行 | 超出 Change Budget，回退 fixer |
| 修改 SPEC.md | 任意行数 | 即使以"优化"为名，也严格禁止 |

## 自查清单（提交报告前必须确认）

- [ ] 我没有跨文件修改代码（涉及 ≥ 2 个文件）
- [ ] 我的直接改动总计 ≤ 5 行（Change Budget 以内）
- [ ] 我没有修改任何接口定义（函数签名、API schema、类型定义）
- [ ] 我没有修改 SPEC.md 或 PRD.md
- [ ] 两份报告（SPEC_Compliance_Check + RealWorld_Risk_Check）均已产出
