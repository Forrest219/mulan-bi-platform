## 1. 背景

当前登录后首页 (`/`) 存在 3 个高优先级交互/文案问题，影响用户明确性和基础体验（问题4已修复）。这些问题属于最小化修复，不涉及架构调整。

---

## 2. 问题描述

### 问题 1: 左下角显示"数据治理"是错误的

**现状**: `ConversationBar` 底部快捷导航只有一个入口"数据治理"（`ri-shield-check-line` 图标，链接到 `/governance/health`）。

**问题**:
- "数据治理"是一级业务导航域的名称，与左侧对话历史栏的定位不符
- 对话历史栏底部应该是系统级操作入口（设置/账户），而不是业务导航
- 语义混淆：用户可能会误以为这是侧边栏的全部导航

**结论**: 底部入口应改为"设置"，作为系统配置的统一入口。

---

### 问题 2: 看不到当前登录账户

**现状**: 登录后首页左侧对话历史栏没有任何用户身份信息展示，用户无法确认自己以什么账号登录，也无法直接退出。

**问题**:
- 登录态不明确，用户可能不确定自己是否成功登录
- 退出登录需要跳转到其他页面或通过浏览器操作，不够便捷
- 违反基础 UX 原则：用户应始终能清楚看到自己的会话状态

**结论**: 在 `ConversationBar` 底部增加用户信息区（显示账号标识 + 角色标签 + 退出登录按钮）。

---

### 问题 3: 左上角隐藏侧边栏按钮应移除

**现状**: `HomeLayout` 在侧边栏折叠时，左上角显示一个小的展开按钮（`ri-arrow-right-s-line` 图标），点击后可重新展开侧边栏。

**问题**:
- 该按钮点击后侧边栏无法恢复（bug），属于严重交互缺陷
- 本次目标是直接移除这个入口，保持侧边栏常驻显示
- 不修复折叠逻辑，直接移除折叠触发入口，是最小成本方案

**结论**: 移除 `HomeLayout` 第82-92行的折叠后展开按钮。

---

## 6. 交互调整说明

### ConversationBar 底部区域改造

**改造前（当前）**:
```
+---------------------------+
| [对话列表...]              |
|                           |
| [新建对话]                 |
+---------------------------+
| [数据治理]                 |  ← 仅1个业务导航入口
+---------------------------+
```

**改造后**:
```
+---------------------------+
| [对话列表...]              |
|                           |
| [新建对话]                 |
+---------------------------+
| [用户头像] 张三 (admin)     |  ← 用户信息行
| [退出登录]                 |  ← 操作入口
+---------------------------+
```

**布局说明**:
- 用户信息行：显示 `display_name` + 角色标签（如 "admin"）
- 退出登录按钮：点击后调用 `logout()`，跳转到登录页
- 不再显示"数据治理"快捷链接

---

### HomeLayout 折叠按钮移除

**移除前**:
- `collapsed=true` 时，左上角出现小按钮，点击可展开侧边栏（bug: 点击后无效果）

**移除后**:
- `collapsed` 状态仍存在（localStorage key 不变），但折叠后不再有UI入口恢复
- 侧边栏始终保持展开状态（符合本次需求）
